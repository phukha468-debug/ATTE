# ЧАСТЬ 0: ПОЗИЦИОНИРОВАНИЕ И ЦЕЛИ ПРОЕКТА

## 0.1 Кто мы

**66ai** — сервис аттестации и повышения ИИ-компетенций сотрудников для малого и среднего бизнеса (МСБ). Мы не продаём тесты. Мы продаём **карту возможностей**: где конкретно в компании зарыто время и как его высвободить с помощью нейросетей.

## 0.2 Миссия

Кратное повышение производительности сотрудников МСБ без найма новых — за счёт автоматизации рутинных процессов с помощью ИИ. Один сотрудник должен мочь выполнять работу двух-трёх, а в отдельных задачах — десяти.

## 0.3 Что мы продаём клиенту

Клиент покупает не «узнать грейд сотрудника», а три вещи:

1. **Карту автоматизации** — конкретный список задач в каждом отделе, которые можно автоматизировать, с расчётом экономии в часах и рублях.
2. **Диагностику навыков** — кто из сотрудников уже умеет использовать ИИ, кому нужно обучение, кто может стать ИИ-чемпионом.
3. **План внедрения** — приоритизированные шаги: что автоматизировать первым, кого обучить, какой ROI ожидать.

## 0.4 Ключевые метрики

| Метрика | Для кого | Суть |
|---------|----------|------|
| Часы экономии на сотрудника/мес | Руководитель | Сколько времени высвобождается |
| Рубли экономии/мес | Руководитель / собственник | Часы × ставку = деньги |
| Грейд сотрудника (1-5) | HR / руководитель | Уровень владения ИИ |
| Adoption rate (через 30 дней) | 66ai | % сотрудников, использующих ИИ ежедневно |

## 0.5 Конкурентное преимущество

Большинство ИИ-тестов проверяют теорию: «Что такое промптинг?», «Назовите модели». Мы проверяем практику: «Покажи, как ты с помощью ИИ за 10 минут сделаешь то, что обычно занимает час». Разница — как между экзаменом ПДД и вождением.

Наш главный актив — **экспертные профили должностей**: для каждой из 20 офисных ролей мы знаем типичную рутину, потенциал автоматизации, эталонные промпты и типичные ошибки. Это экспертиза, упакованная в продукт.

## 0.6 Целевой клиент

Компании МСБ (20-200 сотрудников), где люди работают за компьютером: продажи, маркетинг, HR, финансы, поддержка, операции, юристы, закупки, IT (не разработка), проектное управление, документооборот.

# ЧАСТЬ 1: АРХИТЕКТУРА

## 1.1 Три этапа — воронка, а не три отдельных теста

Этапы работают как последовательная воронка: каждый следующий использует данные предыдущего. Информация накапливается, картина уточняется.

```
ЭТАП 1                    ЭТАП 2                    ЭТАП 3
Карта рутины              Навыки в деле             Микро-проект
─────────────────────────────────────────────────────────────────

Выявляем:                 Проверяем:                Внедряем:
• Какие задачи рутинные   • Умеет ли использовать   • Реальная автоматизация
• Где сотрудник видит       ИИ на практике          • Измеримый результат
  потенциал               • Какое ускорение даёт    • ROI для руководителя
• Базовый уровень         • Где зоны роста
  промптинга              • Валидация карты рутины

     ──────────────>           ──────────────>
     Передаёт:                 Передаёт:
     • Карту рутины            • Баллы по 5 измерениям
     • Баллы по доменам        • Зоны роста
     • Уровень сложности       • Валидированная экономия
       для задания Этапа 2     • Рекомендации для
                                 микро-проекта
```

**Принцип накопления данных:**

Этап 1 говорит: «Сотрудник считает, что может сэкономить 6 ч/нед на КП.»
Этап 2 проверяет: «На практике его ускорение 3x. Реальная экономия — 4 ч/нед.»
Этап 3 внедряет: «Создан шаблон КП. Фактическая экономия за 2 недели — 3.5 ч/нед.»

Итоговый отчёт содержит все три слоя: заявленное → проверенное → внедрённое.

| | Этап 1 | Этап 2 | Этап 3 |
|---|--------|--------|--------|
| Название | Карта рутины | Навыки в деле | Микро-проект |
| Инструмент | Telegram-бот | ИИ-Симулятор | Платформа |
| Время | 15-20 мин | 30-45 мин | 1-2 недели |
| Оценка | Ситуации + LLM-Judge | Dual-AI + LLM-Judge | AI Value Rubric |
| Персонализация | Общая для всех | По профессии + уровню | По карте рутины |

## 1.5 Антиугадывание

Два механизма, применяемых вперемешку:

**Ситуационные сценарии** — рабочие ситуации, все варианты частично правильны, оптимален один.

**Открытые ответы с LLM-Judge** — сотрудник пишет текстом, нейросеть-судья оценивает по рубрике. Минимум 40% вопросов — открытые.

**Ротация заданий (Этап 2)** — на каждый профиль 5-7 заданий, система выдаёт случайное. Сотрудники из одного отдела получают разные задания.

# ЧАСТЬ 3: ЭТАП 2 — НАВЫКИ В ДЕЛЕ (Симулятор)

## 3.1 Суть

Персонализированный симулятор, где сотрудник решает реальную задачу из своей профессии с помощью ИИ. Система оценивает не «знает ли он теорию», а **умеет ли он экономить время на практике**.

Принцип: «Покажи, а не расскажи.» На Этапе 1 сотрудник рассказал о рутине. На Этапе 2 он доказывает, что может ускорить эту рутину с помощью ИИ.

## 3.2 Двойная ИИ-архитектура (Dual-AI)

В симуляторе ИИ играет две раздельные роли, которые не смешиваются:

**ИИ-Ассистент (Sandbox)** — нейтральный помощник. Выполняет запросы сотрудника, как обычный ChatGPT. Не подсказывает, не оценивает, не направляет. Системный промпт: «Ты — ИИ-ассистент. Выполняй запросы. Не подсказывай, не оценивай, не направляй.»

**ИИ-Судья (Judge)** — оценщик, работающий постфактум. Получает полный лог переписки + задание + данные Этапа 1. Анализирует паттерн промптов, качество результата, ускорение. Генерирует развёрнутую обратную связь. НЕ общается с сотрудником.

Почему два, а не один: если ИИ одновременно помогает и оценивает — он невольно направляет сотрудника к лучшим ответам. Мы тестируем реальный навык, а не способность следовать подсказкам.

## 3.3 Поток сотрудника

**Шаг 1 — Выбор профиля.** Сотрудник выбирает направление (11 направлений) → роль (20 ролей). Два уровня: направление (Продажи, Маркетинг, HR…) → конкретная роль (Менеджер B2B, Аккаунт-менеджер…).

**Шаг 2 — Подбор задания.** Система определяет уровень сложности по результатам Этапа 1:

| Результат Этапа 1 | Уровень задания | Характер |
|-------------------|-----------------|----------|
| < 75% или промптинг < 60% | Базовый | Одна задача, 1-2 промпта |
| ≥ 75%, промптинг ≥ 60% | Средний | 2-3 итерации, развёрнутый бриф |
| ≥ 85%, промптинг ≥ 75% | Продвинутый | Сложный бриф, пайплайн из шагов |

Задание подбирается близко к задачам из карты рутины Этапа 1. Если продажник написал, что тратит 2 часа на КП — он получит задачу на составление КП.

Из пула соответствующего уровня выбирается случайное задание (ротация — сотрудники из одного отдела получают разные задания).

**Шаг 3 — Sandbox.** Сотрудник работает с ИИ-ассистентом: пишет промпты, получает результаты, дорабатывает. Таймер работает. Все промпты записываются. Лимит: 45 минут.

**Шаг 4 — Завершение.** Сотрудник нажимает «Готово», загружает финальный результат. Таймер останавливается.

**Шаг 5 — Оценка Judge.** Автоматическая оценка по 5 измерениям + генерация обратной связи.

**Шаг 6 — Обратная связь.** Сотрудник получает не просто баллы, а развёрнутый разбор: что было хорошо, где потерял время, как можно было сделать эффективнее — с примерами лучших промптов для его задачи.

## 3.4 Модульная система скиллов

Архитектура построена на разделении универсального и специфичного:

**Core Skill (Ядро Judge)** — одно на все профессии. Содержит:
- 5 измерений оценки с рубриками (1-5)
- Формулу итогового балла
- Шаблон обратной связи
- Правила интеграции с Этапом 1
- Логику вызова профильных модулей

**Profile Module (Экспертный профиль)** — по одному на роль (20 штук). Содержит:
- Типичная рутина должности с хронометражем (экспертная карта)
- ТОП-3 задачи для автоматизации (подсвечиваем клиенту)
- Эталонные промпты-образцы (для обратной связи)
- Типичные ошибки данной профессии при работе с ИИ
- Профильные критерии качества результата
- Метаданные профиля

**Task Pool (Пул заданий)** — по одному на роль. Содержит:
- 5-7 заданий трёх уровней сложности (базовый, средний, продвинутый)
- Полный бриф/ТЗ для каждого задания
- Бенчмарки ручной работы
- Специфические критерии для Judge

Почему модульная:
- Обновил критерий промптов → правишь 1 файл (Core), а не 20
- Новый профиль = 1 профиль + 1 пул заданий, Core не трогаем
- После стресс-тестов калибруешь модули независимо

## 3.5 Пять измерений оценки (Judge)

| # | Измерение | Вес | Что оценивает |
|---|-----------|-----|---------------|
| 1 | Качество промптинга | 30% | Контекст, роль, формат, ограничения — по всей цепочке промптов |
| 2 | Итеративность | 20% | Доработал ли результат? Принял первый ответ или улучшал? |
| 3 | Ускорение | 20% | Бенчмарк / фактическое время (с поправкой на качество) |
| 4 | Качество результата | 20% | Профильные критерии (для продажника — одни, для HR — другие) |
| 5 | Понимание ограничений ИИ | 10% | Не принял ли галлюцинации за факт? Проверил ли? |

Формула: Итого = (П × 0.30) + (И × 0.20) + (У × 0.20) + (К × 0.20) + (О × 0.10)

Шкала каждого измерения: 1-5 (с детальными рубриками в Core Skill).

Итоговый порог Этапа 2: Промпты ≥ 3.5/5 + Ускорение ≥ 3x + Качество принято.

## 3.6 Обратная связь — главный продукт Этапа 2

Сотрудник получает не «3.8 из 5», а развёрнутый разбор:

- **Анализ каждого промпта:** что хорошо, что улучшить, пример лучшего промпта
- **Главная зона роста:** 1-2 проблемы + конкретные решения
- **Валидация карты рутины:** «Вы указали 6 ч/нед на КП. Ваше ускорение 4x. Реальная экономия: ~4.5 ч/нед = ~27 000 ₽/мес»
- **Рекомендации:** что делать завтра на работе (для грейда 1-2: обучающий модуль; для 3: техники промптинга; для 4-5: масштабирование)

## 3.7 Экспертные профили — главный актив 66ai

Для каждой из 20 ролей мы формируем экспертный профиль: «Вот типичная рутина этой должности, вот что чаще всего можно автоматизировать, вот какое ускорение реалистично, вот эталонные промпты.»

Профили создают три слоя ценности:

**Для сотрудника:** понимание, какие его задачи можно автоматизировать, с примерами промптов.
**Для руководителя:** карта автоматизации отдела с расчётом экономии.
**Для 66ai:** актив, улучшающийся с каждым клиентом — конкурентное преимущество.

Отчёты содержат данные ВСЕХ трёх этапов — показывая накопление: заявленное (Этап 1) → проверенное (Этап 2) → внедрённое (Этап 3).

**По сотруднику:** грейд + баллы по всем этапам + карта рутины + валидированная экономия + зоны роста + рекомендации.

**По руководителю:** всё из сотрудника + карта отдела + потенциал отдела + стратегия внедрения.

**Сводный по компании:** карта автоматизации компании + рейтинг отделов + общий потенциал (часы/мес, ₽/мес, ₽/год) + приоритизированный план внедрения + ROI.

# ЧАСТЬ 7: КАЛИБРОВКА И СТРЕСС-ТЕСТЫ

## 7.1 План калибровки

10 бесплатных компаний — стресс-тест вместо оплаты. После каждой компании — интервью и итерация:

**До теста:** состав отделов, роли, ожидания руководителя.
**Во время:** логи Sandbox, время, результаты Judge.
**После (интервью):**
- «Задание было реалистичным?» → калибровка заданий
- «Обратная связь была полезной?» → калибровка Judge
- «Что в вашей работе мы не покрыли?» → дополнение профиля
- «Какие задачи вы хотели бы автоматизировать, но не знали как?» → расширение ТОП-3

## 7.2 Итерационный цикл

```
Компания 1 → интервью → правки профилей и Judge →
Компания 2 → интервью → правки →
...
Компания 10 → финальная версия профилей
```

Не «провели 10, потом поправили», а «провели 1 → поправили → провели 2 → поправили». К десятой компании профили отполированы.

## 7.3 Целевые метрики калибровки

- Средний балл без подготовки: 40-50% (если выше — тест слишком лёгкий)
- После обучения: 70-80%
- Обратная связь «полезна»: ≥ 80% сотрудников
- Задание «реалистично»: ≥ 90% сотрудников
