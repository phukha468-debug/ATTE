## 2. ТОП-3 ЗАДАЧИ ДЛЯ АВТОМАТИЗАЦИИ

### 2.1 Описания вакансий + скрипты собеседований
- **Почему #1:** Высокая частота + текстовый формат + повторяемость шаблонов
- **Что делает ИИ:** Генерирует структурированную вакансию с ЕVP, подбирает вопросы с критериями оценки
- **Что остаётся человеку:** Проверка требований с нанимающим менеджером, адаптация тона
- **Ускорение-ориентир:** 5-10x

### 2.2 Коммуникации с кандидатами (письма)
- **Почему #2:** Ежедневная задача, 80% писем типовые, высокая повторяемость
- **Что делает ИИ:** Генерирует персонализированные письма по шаблонам (приглашение, отказ, оффер, follow-up)
- **Что остаётся человеку:** Персональные детали, проверка имён/дат
- **Ускорение-ориентир:** 8-15x

### 2.3 Адаптация и onboarding
- **Почему #3:** Высокая трудоёмкость, масштабируемость на всю компанию
- **Что делает ИИ:** Генерирует план адаптации по должности, чек-листы, welcome-письма
- **Что остаётся человеку:** Согласование с руководителем, назначение наставника
- **Ускорение-ориентир:** 5-8x

## 3. ЭТАЛОННЫЕ ПРОМПТЫ-ОБРАЗЦЫ

### Эталон 1: Описание вакансии

```
Ты — IT-рекрутер с 5-летним опытом подбора в технологические компании.

Задача: составь описание вакансии «Менеджер по продажам».

Контекст компании:
- Компания: IT-дистрибьютор серверного оборудования, Москва
- Размер: 50 сотрудников, растут на 30% в год
- Культура: молодая команда, плоская иерархия, гибрид (2 дня офис / 3 удалённо)
- EVP: обучение за счёт компании, бонусы до 3 окладов/год, ДМС с первого дня

Требования от нанимающего менеджера:
- Опыт B2B продаж от 2 лет (IT — плюс, не обязательно)
- Умение вести сложные переговоры
- CRM (любая), Excel — уверенный
- Зарплата: 100-150 тыс + бонус

Структура вакансии:
1. Цепляющий заголовок (не «Менеджер по продажам», а с акцентом на ценность)
2. О компании (3-4 предложения, без «динамично развивающаяся»)
3. Чем предстоит заниматься (5-7 пунктов, глаголами)
4. Что мы ждём (5-6 требований, разделить must/nice-to-have)
5. Что предлагаем (EVP, конкретные бенефиты с цифрами)
6. Условия (формат, график, зарплата)

Тон: человечный, конкретный. Без канцеляризмов и «стрессоустойчивость».
```

### Эталон 2: Скрипт собеседования

```
Составь скрипт телефонного скрининга (15 мин) для вакансии
«Менеджер по продажам» в IT-дистрибьюторе.

Контекст: кандидат откликнулся на вакансию, резюме подходит по формальным
критериям. Цель звонка: проверить мотивацию, базовые hard/soft skills,
зарплатные ожидания, готовность к формату работы.

Дай 10 вопросов:
- 3 на мотивацию (почему сменить работу, почему наша отрасль)
- 3 на hard skills (опыт B2B, работа с CRM, цикл сделки)
- 2 на soft skills (переговоры, работа с возражениями)
- 2 на условия (зарплата, формат, сроки выхода)

Для каждого вопроса дай:
- Формулировку вопроса
- Что слушаем (на что обращать внимание в ответе)
- Зелёный флаг (хороший ответ)
- Красный флаг (тревожный сигнал)

Формат: таблица. Тон вопросов: дружелюбный, но структурированный.
```

### Эталон 3: Письмо-отказ

```
Напиши профессиональное письмо-отказ кандидату после собеседования.

Контекст:
- Кандидат: Мария, 28 лет, junior маркетолог
- Собеседование было на позицию «маркетолог» в IT-компании
- Причина отказа: недостаточно опыта в B2B (нужен от 2 лет, у Марии — 1 год B2C)
- Впечатление: хорошая коммуникация, потенциал, но сейчас не подходит
- Хотим сохранить в кадровом резерве

Требования к письму:
- Не обижать и не обесценивать
- Конкретно сказать причину (не «мы выбрали другого кандидата»)
- Предложить остаться в кадровом резерве
- Длина: 100-150 слов
- Тон: тёплый, уважительный, честный
```

## 4. ТИПИЧНЫЕ ОШИБКИ HR ПРИ РАБОТЕ С ИИ

| # | Ошибка | Пример | Что делать |
|---|--------|--------|-----------|
| 1 | Шаблонные вакансии | «Напиши вакансию менеджера» — без EVP и культуры | Добавить: компания, EVP, тон, конкретные бенефиты |
| 2 | Генерация ПД кандидатов | Загрузка резюме с контактами в бесплатный ChatGPT | Использовать корпоративный API или анонимизировать |
| 3 | Юридические галлюцинации | ИИ написал «по ТК РФ испытательный срок 6 месяцев» (на самом деле 3) | ВСЕГДА проверять юридические нормы вручную |
| 4 | Нет критериев оценки | Скрипт собеседования без «что слушаем» | Просить критерии для каждого вопроса |
| 5 | Копипаст отказов | Один шаблон на всех кандидатов | Добавлять контекст: причина, что понравилось |
| 6 | Нет адаптации под уровень | Один скрипт для junior и senior | Указывать уровень позиции в промпте |
| 7 | Доверие к оценке soft skills | ИИ «оценил» кандидата по резюме | ИИ не может оценивать людей, только структурировать данные |

## 6. ПРОФИЛЬНЫЕ КРИТЕРИИ КАЧЕСТВА РЕЗУЛЬТАТА

### Для задачи «Вакансия»:
- Структура полная (все блоки): 0-2
- EVP конкретный (цифры, бенефиты, не «дружный коллектив»): 0-2
- Тон человечный (нет канцеляризмов): 0-2
- Требования разделены на must/nice-to-have: 0-2
- Адаптировано под отрасль и компанию: 0-2
- **Максимум: 10**

### Для задачи «Скрипт собеседования»:
- Вопросы покрывают hard + soft + мотивацию: 0-2
- Есть критерии оценки ответов: 0-3
- Вопросы адаптированы под уровень позиции: 0-2
- Формулировки не «допросные»: 0-1
- Реалистичны для 15-минутного скрининга: 0-2
- **Максимум: 10**

### Для задачи «Адаптационный план»:
- Покрывает первые 30/60/90 дней: 0-3
- Конкретные шаги, а не абстракции: 0-3
- Учтена специфика должности: 0-2
- Есть метрики успешной адаптации: 0-2
- **Максимум: 10**
