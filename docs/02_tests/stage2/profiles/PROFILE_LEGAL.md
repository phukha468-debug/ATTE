## 2. ТОП-3 ЗАДАЧИ ДЛЯ АВТОМАТИЗАЦИИ

### 2.1 Проекты типовых договоров
- **Почему #1:** 60-70% договоров МСБ — типовые (поставка, услуги, аренда, подряд). ИИ генерирует первый драфт мгновенно
- **Что делает ИИ:** Генерирует структуру + наполнение по вводным (стороны, предмет, сроки, сумма, ответственность)
- **Что остаётся человеку:** Проверка соответствия закону, адаптация под конкретные риски, согласование
- **Ускорение-ориентир:** 3-5x

### 2.2 Разъяснения и справки для сотрудников / руководства
- **Почему #2:** Ежедневный поток «а можно ли нам так сделать?» — ИИ помогает структурировать ответ
- **Что делает ИИ:** Формулирует правовую позицию понятным языком, подбирает аргументы
- **Что остаётся человеку:** Проверка норм, учёт судебной практики, финальная формулировка
- **Ускорение-ориентир:** 5-10x

### 2.3 Регламенты и внутренние политики
- **Почему #3:** Создание с нуля — долго, ИИ отлично структурирует типовые документы
- **Что делает ИИ:** Генерирует структуру, наполняет по контексту компании, адаптирует формулировки
- **Что остаётся человеку:** Проверка юридической корректности, согласование с руководством
- **Ускорение-ориентир:** 5-8x

## 3. ЭТАЛОННЫЕ ПРОМПТЫ-ОБРАЗЦЫ

### Эталон 1: Проект договора оказания услуг

```
Ты — юрисконсульт IT-компании (разработка на заказ).

Задача: подготовь проект договора на оказание услуг по разработке 
мобильного приложения.

Данные:
- Исполнитель: ООО «КодМастер» (наша компания)
- Заказчик: ООО «ФудПроф» (доставка еды для офисов)
- Предмет: разработка мобильного приложения для заказа еды (iOS + Android)
- Стоимость: 2 400 000 ₽ (фикс, поэтапная оплата)
- Сроки: 4 месяца от подписания
- Этапы: ТЗ (2 нед) → Дизайн (3 нед) → Разработка (8 нед) → Тестирование (3 нед)
- Оплата: 30% аванс, 30% после дизайна, 40% после приёмки
- Наши риски: заказчик может затянуть согласование макетов
- Гарантия: 3 месяца на исправление багов после сдачи
- ИП: исключительные права переходят заказчику после полной оплаты

Структура договора:
1. Предмет
2. Права и обязанности сторон
3. Стоимость и порядок оплаты
4. Сроки и этапы
5. Порядок приёмки (акт + сроки рассмотрения)
6. Интеллектуальная собственность
7. Конфиденциальность
8. Ответственность сторон
9. Форс-мажор
10. Разрешение споров
11. Заключительные положения
12. Реквизиты сторон

Важно: 
- Включить пункт о приёмке: если заказчик не подписал акт 
  и не предоставил мотивированный отказ в течение 5 рабочих дней —
  работы считаются принятыми.
- Включить ограничение ответственности (не более суммы договора).
- НЕ ссылайся на конкретные статьи ГК — я проверю и добавлю сам.
  Вместо этого пиши: «в соответствии с действующим законодательством РФ».
```

### Эталон 2: Юридическая справка для руководителя

```
Руководитель спрашивает: «Мы хотим начать собирать email-адреса 
посетителей сайта для рассылки. Что нам нужно юридически?»

Подготовь справку (понятным языком, не юридическим):

Контекст:
- Компания: интернет-магазин товаров для дома
- Сайт: собирает email через форму «подпишитесь на скидки»
- Планируют: еженедельная промо-рассылка
- В РФ: нужно учитывать ФЗ-152 (персональные данные) и ФЗ-38 (реклама)

Структура справки:
1. Короткий ответ (2-3 предложения — «что нужно сделать»)
2. Подробнее: какие документы нужны (политика обработки ПД, согласие)
3. Что разместить на сайте (чек-лист)
4. Риски при нарушении (штрафы — без конкретных сумм, я проверю)
5. Рекомендация: минимальный план действий

Тон: как объясняешь коллеге, не как юридическое заключение.
Длина: 1 страница. НЕ ссылайся на конкретные статьи ФЗ — 
я проверю актуальность сам.
```

### Эталон 3: Претензионное письмо

```
Подготовь досудебную претензию контрагенту.

Ситуация:
- Наша компания: ООО «РемонтСервис» (ремонт офисов)
- Контрагент: ООО «МебельГрупп» (поставщик мебели)
- Договор: поставка офисной мебели, №45 от 10.02.2026
- Сумма: 850 000 ₽ (оплачен аванс 100% — 12.02.2026)
- Срок поставки по договору: 15.03.2026
- Факт: на 16.04.2026 мебель не поставлена, контрагент не выходит на связь
- Наш ущерб: задержка сдачи объекта клиенту, неустойка нашему клиенту ~200 000 ₽

Что требуем:
1. Поставить мебель в течение 7 календарных дней
2. ИЛИ вернуть аванс 850 000 ₽ + неустойку за просрочку

Тон: строгий, деловой, без эмоций. Угроза суда — если не исполнят 
в срок. Длина: 1 страница. НЕ ссылайся на конкретные статьи ГК — 
напиши «в соответствии с условиями договора и действующим 
законодательством», я добавлю нормы сам.
```

## 4. ТИПИЧНЫЕ ОШИБКИ ЮРИСТОВ ПРИ РАБОТЕ С ИИ

| # | Ошибка | Пример | Что делать |
|---|--------|--------|-----------|
| 1 | Доверие к нормативным ссылкам | ИИ написал «ст. 782 ГК» — статья существует, но не та | НИКОГДА не брать ссылки из ИИ без проверки |
| 2 | Устаревшие нормы | ИИ ссылается на редакцию закона 2019 года | Проверять актуальность через КонсультантПлюс / Гарант |
| 3 | Загрузка конфиденциального | Вставил договор с суммами и реквизитами в ChatGPT | Корпоративный API, обезличить данные |
| 4 | Выдуманная судебная практика | ИИ «нашёл» дело №А40-12345/2024 — его не существует | ИИ фабрикует номера дел; проверять через КАД |
| 5 | Универсальные формулировки | «Стороны несут ответственность» — без конкретики | Указать: что именно (неустойка %, лимит, сроки) |
| 6 | Без контекста рисков | Договор без защитных пунктов | Указать: наши риски, что нужно защитить |

## 5. СВЯЗКА С ЭТАПОМ 1

```
ЕСЛИ >= 85% И промптинг >= 75%: → Продвинутый (договор + правовой аудит)
ЕСЛИ >= 75% И промптинг >= 60%: → Средний (договор / справка / претензия)
ЕСЛИ < 75% ИЛИ промптинг < 60%: → Базовый (одно письмо / один документ)
```

## 6. ПРОФИЛЬНЫЕ КРИТЕРИИ КАЧЕСТВА РЕЗУЛЬТАТА

### Для задачи «Договор»:
- Структура полная (все ключевые разделы): 0-2
- Предмет конкретизирован (не generic): 0-2
- Есть защитные механизмы (неустойка, приёмка, ограничение ответственности): 0-3
- Формулировки юридически корректны: 0-2
- Нет выдуманных нормативных ссылок: 0-1
- **Максимум: 10**

### Для задачи «Справка / заключение»:
- Ответ на вопрос руководителя дан чётко: 0-3
- Понятен не-юристу: 0-2
- Есть практические рекомендации (не только «по закону»): 0-3
- Указаны риски: 0-2
- **Максимум: 10**

### Для задачи «Претензия / письмо»:
- Структура: факты → нарушение → требование → последствия: 0-3
- Тон деловой, без эмоций: 0-2
- Требования конкретны (сумма, срок): 0-3
- Нет выдуманных ссылок на нормы: 0-2
- **Максимум: 10**
