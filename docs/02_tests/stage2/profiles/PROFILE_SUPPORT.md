## 2. ТОП-3 ЗАДАЧИ ДЛЯ АВТОМАТИЗАЦИИ

### 2.1 База ответов на типовые вопросы (FAQ как основа)
- **Почему #1:** 80% вопросов повторяются. Одна хорошая FAQ-база экономит часы каждый день каждому специалисту
- **Что делает ИИ:** Генерирует ответы по шаблонам бренда, адаптирует под тон, создаёт варианты для разных каналов (email, чат, телефон)
- **Что остаётся человеку:** Проверка фактов, персонализация под конкретного клиента, эмпатия в сложных случаях
- **Ускорение-ориентир:** 8-15x

### 2.2 Разбор жалоб и формулировка ответов
- **Почему #2:** Эмоционально тяжёлая задача, ИИ помогает отстраниться и сформулировать корректно
- **Что делает ИИ:** Структурирует ответ (признание проблемы → решение → компенсация → извинение), подбирает правильный тон
- **Что остаётся человеку:** Решение по компенсации, эмпатия, принятие ответственности
- **Ускорение-ориентир:** 3-5x

### 2.3 Классификация и маршрутизация тикетов
- **Почему #3:** Занимает много времени, ИИ справляется отлично при чётких правилах
- **Что делает ИИ:** Классифицирует по теме/срочности/отделу, формулирует сводку для ответственного
- **Что остаётся человеку:** Проверка спорных случаев, эскалация VIP-клиентов
- **Ускорение-ориентир:** 3-5x

## 3. ЭТАЛОННЫЕ ПРОМПТЫ-ОБРАЗЦЫ

### Эталон 1: Ответ на типовой вопрос

```
Ты — специалист службы поддержки интернет-магазина товаров для дома.

Клиент написал: «Заказала набор кастрюль 3 дня назад (заказ #58291),
обещали доставить за 2 дня. До сих пор не пришёл. Что происходит?»

Данные по заказу:
- Статус: передан курьеру 2 дня назад
- Причина задержки: ошибка сортировочного центра, задержка 1 день
- Новый срок доставки: завтра до 18:00
- Компенсация, которую мы можем предложить: промокод 500 ₽ на следующий заказ

Напиши ответ клиенту:
- Извинись (искренне, не формально)
- Объясни причину (честно, без корпоративного языка)
- Дай конкретный новый срок
- Предложи компенсацию
- Заверь, что следишь за доставкой лично

Тон: тёплый, уважительный. Длина: 100-150 слов. Без шаблонных фраз типа 
«приносим извинения за доставленные неудобства».
```

### Эталон 2: Создание FAQ по продукту

```
Ты — специалист поддержки SaaS-сервиса облачной телефонии для бизнеса.

Задача: создай раздел FAQ по теме «Настройка записи разговоров».

Для раздела подготовь 8 вопросов, закрывающих весь путь пользователя:
- От базового («Как включить запись?») 
- До продвинутого («Как настроить запись только для определённых менеджеров?»)

Для каждого вопроса дай:
- Формулировку вопроса (как написал бы реальный пользователь, 
  естественно, с возможными ошибками/разговорным стилем)
- Короткий ответ (1-2 предложения для быстрого понимания)
- Развёрнутый ответ (3-5 пунктов с инструкцией)
- Связанные вопросы (ссылки на смежные темы FAQ)
- Если есть ограничения — указать (например, «доступно только в тарифе Pro»)

Тон: дружелюбный, не перегруженный техническим жаргоном. 
Аудитория: собственники МСБ без IT-подготовки.
```

### Эталон 3: Ответ на жалобу (эмоциональный случай)

```
Ты — менеджер службы поддержки банка.

Клиент написал гневное письмо:
«Ваш банк — катастрофа! Перевёл деньги третий день, до сих пор не дошли. 
Звонил 5 раз в поддержку — каждый раз разные операторы, все обещают 
перезвонить и никто не звонит. Я теряю деньги из-за просрочки платежа 
поставщику. Если деньги не вернутся сегодня — буду писать в ЦБ!»

Данные:
- Перевод действительно завис из-за технического сбоя в платёжной системе
- Проблему устранили, деньги вернутся на счёт в течение 2 часов
- Клиенту действительно не перезванивали (ошибка координации в поддержке)
- Мы можем компенсировать: возврат комиссии + 1 месяц бесплатного обслуживания
- У клиента VIP-статус, в банке 4 года

Напиши ответ:
- Признай ошибки (обе: технический сбой + отсутствие обратной связи)
- Дай конкретный срок решения (по часам)
- Предложи компенсацию
- Выдели личную линию связи с тобой (не общая поддержка)
- Не оправдывайся, не обещай того, что не можешь гарантировать

Тон: профессиональный, ответственный, без заискивания. 
Длина: 150-200 слов.
```

## 4. ТИПИЧНЫЕ ОШИБКИ СПЕЦИАЛИСТОВ ПОДДЕРЖКИ ПРИ РАБОТЕ С ИИ

| # | Ошибка | Пример | Что делать |
|---|--------|--------|-----------|
| 1 | Шаблонные ответы | Копипаст ответа ИИ без персонализации | Добавить в промпт имя клиента, номер заказа, историю |
| 2 | Нет эмпатии в промпте | «Напиши ответ клиенту с жалобой» | Добавить: «признай проблему, извинись искренне» |
| 3 | ИИ-галлюцинации в ответах | «Ваш заказ будет доставлен согласно п. 4.2 договора» — пункта нет | Проверять все ссылки на документы и пункты |
| 4 | Потеря данных клиента | Загрузка номеров карт / паспортов в бесплатный ChatGPT | Использовать корпоративный API, маскировать ПД |
| 5 | Нет контекста канала | Длинный email-ответ в чат на 5 строк | Указывать канал и его ограничения |
| 6 | Обещания от лица компании | ИИ пишет «мы вернём деньги в течение дня» без полномочий | Промпт должен указывать границы: «не обещай возврат/компенсацию» |
| 7 | Игнорирование истории | Отвечает как на первое обращение, хотя клиент писал 3 раза | Передавать в промпт историю тикета |

## 6. ПРОФИЛЬНЫЕ КРИТЕРИИ КАЧЕСТВА РЕЗУЛЬТАТА

### Для задачи «Ответ клиенту»:
- Персонализация (имя, детали заказа/проблемы): 0-2
- Есть эмпатия / признание проблемы: 0-2
- Конкретное решение / срок: 0-3
- Тон соответствует ситуации (не шаблонный): 0-2
- Длина адекватная каналу: 0-1
- **Максимум: 10**

### Для задачи «FAQ / база знаний»:
- Покрывает путь пользователя (базовые → продвинутые): 0-3
- Формулировки вопросов естественные (как пишут реальные пользователи): 0-2
- Ответы структурированы (короткий + развёрнутый): 0-2
- Тон дружелюбный, без жаргона: 0-1
- Есть связи между вопросами: 0-2
- **Максимум: 10**

### Для задачи «Разбор жалобы»:
- Признание проблемы (не обесценивание): 0-2
- Честное объяснение (без корпоративного языка): 0-2
- Конкретный план решения с сроками: 0-3
- Адекватная компенсация: 0-2
- Тон профессиональный, без заискивания: 0-1
- **Максимум: 10**
