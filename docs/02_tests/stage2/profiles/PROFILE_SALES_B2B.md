## 2. ТОП-3 ЗАДАЧИ ДЛЯ АВТОМАТИЗАЦИИ (подсвечиваем клиенту)

Эти задачи дают максимальный ROI и должны быть в фокусе обратной связи.

### 2.1 Коммерческие предложения из ТЗ клиента
- **Почему #1:** Высокая частота × долгое время × высокое качество ИИ-результата
- **Что делает ИИ:** Генерирует структуру КП, описание решения, этапы, сроки, УТП — по шаблону компании и ТЗ клиента
- **Что остаётся человеку:** Проверка цен, согласование скидок, финальная адаптация под конкретные договорённости
- **Типичный промпт-паттерн:** Роль + ТЗ клиента + шаблон КП + ограничения (бюджет, сроки) + тон
- **Ускорение-ориентир:** С 60 мин до 8-12 мин (5-8x)

### 2.2 Follow-up и email-цепочки
- **Почему #2:** Ежедневная задача, высокая повторяемость, текстовый формат
- **Что делает ИИ:** Генерирует персонализированные письма по контексту переговоров
- **Что остаётся человеку:** Проверка фактов, добавление личных деталей
- **Типичный промпт-паттерн:** Контекст встречи/звонка + цель письма + тон + CTA
- **Ускорение-ориентир:** С 20 мин до 2-3 мин (8-10x)

### 2.3 Скрипты и обработка возражений
- **Почему #3:** Масштабируемость — один хороший скрипт используется всем отделом
- **Что делает ИИ:** Генерирует варианты ответов на типовые возражения, адаптирует под продукт и отрасль
- **Что остаётся человеку:** Тестирование на реальных звонках, доработка по результатам
- **Типичный промпт-паттерн:** Продукт + портрет клиента + список возражений + формат (вопрос-ответ)
- **Ускорение-ориентир:** С 45 мин до 7-10 мин (5-6x)

## 3. ЭТАЛОННЫЕ ПРОМПТЫ-ОБРАЗЦЫ

Используются Judge для сравнения и в обратной связи — как пример «так надо».

### Эталон 1: КП из ТЗ

```
Ты — опытный менеджер по B2B-продажам IT-решений для среднего бизнеса.

Задача: составь коммерческое предложение на внедрение CRM-системы.

Данные клиента:
- Компания: ООО «ПромТех», оптовая торговля стройматериалами, 80 сотрудников
- Запрос: автоматизация отдела продаж (12 менеджеров), интеграция с 1С
- Бюджет: до 500 000 ₽
- Срок: 2 месяца
- Боль: менеджеры ведут клиентов в Excel, теряются заявки

Структура КП:
1. Титульный лист (название, дата, получатель)
2. Понимание задачи клиента (покажи, что понял их боль)
3. Предлагаемое решение (конкретная CRM + почему она подходит)
4. Этапы внедрения с таймлайном
5. Стоимость с разбивкой по этапам (уложись в 500 000 ₽)
6. Гарантии и поддержка
7. Следующий шаг (призыв к действию)

Тон: профессиональный, но не канцелярский. Конкретные цифры, без воды.
Объём: 2-3 страницы.
```

### Эталон 2: Follow-up после встречи

```
Напиши follow-up письмо клиенту после первой встречи.

Контекст:
- Клиент: Андрей Викторович, коммерческий директор «ЛогиСтар» (логистика, 40 чел.)
- Встреча была вчера, длилась 45 мин
- Обсудили: проблемы с учётом заявок, потребность в CRM, он сравнивает нас с двумя конкурентами
- Он попросил прислать кейсы из логистики и детальный расчёт
- Следующий шаг: согласовать демо на следующую неделю

Задача письма: закрепить договорённости, показать экспертизу, продвинуть к демо.
Тон: дружелюбно-деловой. Не подобострастный.
Длина: 150-200 слов.
```

### Эталон 3: Обработка возражений

```
Ты — тренер отдела продаж в IT-компании, которая продаёт CRM-системы.

Составь скрипт обработки 5 самых частых возражений при B2B-продаже CRM:
1. «Нам это дорого»
2. «Мы уже пробовали CRM, не прижилась»
3. «У нас и в Excel всё работает»
4. «Нет времени на внедрение»
5. «Решение принимает директор, а не я»

Для каждого возражения дай:
- Что на самом деле имеет в виду клиент (истинная причина)
- Ответ менеджера (2-3 варианта разной интенсивности)
- Вопрос, который переводит разговор дальше

Формат: таблица. Язык: живой, не шаблонный. Для среднего бизнеса (не корпорации).
```

## 4. ТИПИЧНЫЕ ОШИБКИ ПРОДАЖНИКОВ ПРИ РАБОТЕ С ИИ

Judge использует этот список для диагностики и обратной связи.

| # | Ошибка | Пример | Почему плохо | Что делать |
|---|--------|--------|-------------|-----------|
| 1 | Нулевой контекст | «Напиши КП» | Получит generic шаблон, бесполезный для клиента | Добавить: кто клиент, его боль, бюджет, продукт |
| 2 | Нет итераций | Принял первый ответ как финальный | КП/письмо содержит неточности, нет персонализации | Минимум 2-3 итерации: сгенерировал → проверил → попросил доработать |
| 3 | Копипаст без проверки | Отправил клиенту текст из ИИ «как есть» | Галлюцинации в ценах, сроках, юридических нормах | Проверять ВСЕ факты, цифры и обещания |
| 4 | Нет формата результата | «Сделай анализ конкурентов» | ИИ не знает, нужна таблица или текст, сколько конкурентов | Указать: формат, объём, структуру, для кого результат |
| 5 | Не использует роль | Обращается к ИИ как к «поисковику» | Ответы поверхностные, без экспертизы | Задать роль: «Ты — менеджер по продажам с 10-летним опытом в IT» |
| 6 | Просит запрещённое | «Дай базу email-ов компаний для холодной рассылки» | ИИ не имеет доступа к базам, результат — галлюцинация | Понимать, что ИИ генерирует текст, а не даёт реальные данные |
| 7 | Не масштабирует | Написал КП вручную с помощью ИИ, но не сохранил промпт | Следующее КП придётся делать заново | Сохранять успешные промпты как шаблоны |

## 5. СВЯЗКА С ЭТАПОМ 1

### Что мы получаем из Этапа 1 для этого профиля:

**Из карты рутины (вопрос A2):** 5 задач, которые сотрудник сам назвал рутинными. Для продажника типично: КП, письма, CRM, отчёты, подготовка к встречам.

**Из открытых ответов (A1, A5):** Как сотрудник думает об автоматизации — наивно или зрело.

**Из баллов за промптинг (B1, B3):** Базовый уровень промптинга — этот балл определяет уровень задания на Этапе 2.

### Логика подбора задания:

```
ЕСЛИ балл_этап1 >= 85% И промптинг >= 75%:
    → Задание уровня «Продвинутый» (сложный бриф, пайплайн)
ЕСЛИ балл_этап1 >= 75% И промптинг >= 60%:
    → Задание уровня «Средний» (стандартная задача, 2-3 итерации)
ЕСЛИ балл_этап1 < 75% ИЛИ промптинг < 60%:
    → Задание уровня «Базовый» (одна простая задача)
```

### Валидация карты рутины:

Если сотрудник на Этапе 1 написал «я трачу 4 часа на КП» — на Этапе 2 он получит задачу на КП. Если он не справится за разумное время или покажет слабый промптинг — значит, его оценка потенциала экономии была завышена. Это войдёт в отчёт.

## 7. ПРОФИЛЬНЫЕ КРИТЕРИИ КАЧЕСТВА РЕЗУЛЬТАТА

Эти критерии специфичны для продажника. Judge использует их поверх общих критериев промптинга.

### Для задачи «КП»:
- Есть ли понимание боли клиента (не generic, а конкретное): 0-3
- Структура КП полная (все блоки на месте): 0-2
- Стоимость реалистична и в рамках бюджета: 0-2
- Есть CTA (призыв к действию): 0-1
- Тон соответствует B2B (не продаёт агрессивно, не слишком формально): 0-2
- **Максимум: 10**

### Для задачи «Follow-up»:
- Есть привязка к контексту встречи/звонка: 0-3
- Зафиксированы конкретные договорённости: 0-2
- Есть следующий шаг с датой: 0-2
- Длина адекватная (не роман, не две строчки): 0-1
- Тон: профессиональный, но человечный: 0-2
- **Максимум: 10**

### Для задачи «Скрипт возражений»:
- Возражения раскрыты (не поверхностно): 0-3
- Ответы реалистичные, а не из учебника: 0-3
- Есть переходные вопросы: 0-2
- Адаптировано под продукт/отрасль: 0-2
- **Максимум: 10**
