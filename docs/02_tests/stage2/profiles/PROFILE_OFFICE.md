## 2. ТОП-3 ЗАДАЧИ ДЛЯ АВТОМАТИЗАЦИИ

### 2.1 Протоколы совещаний
- **Почему #1:** Высокая частота + трудоёмкость + чёткий формат. Идеальная задача для ИИ
- **Что делает ИИ:** По заметкам / записи формирует структурированный протокол: решения, ответственные, сроки
- **Что остаётся человеку:** Проверка фактов, уточнение формулировок с участниками
- **Ускорение-ориентир:** 5-10x

### 2.2 Деловая переписка и внутренние коммуникации
- **Почему #2:** Ежедневный объём, 80% — типовые форматы
- **Что делает ИИ:** Генерирует письма, объявления, напоминания, поздравления по контексту
- **Что остаётся человеку:** Персонализация, проверка деталей, тон
- **Ускорение-ориентир:** 5-10x

### 2.3 Регламенты и внутренние документы
- **Почему #3:** Создание с нуля — долго, ИИ отлично структурирует
- **Что делает ИИ:** Генерирует структуру документа, наполняет по вводным, форматирует
- **Что остаётся человеку:** Проверка с юристом/руководителем, адаптация под реалии
- **Ускорение-ориентир:** 5-8x

## 3. ЭТАЛОННЫЕ ПРОМПТЫ-ОБРАЗЦЫ

### Эталон 1: Протокол совещания

```
Ты — ассистент руководителя в IT-компании (60 человек).

Задача: оформи протокол совещания по моим заметкам.

Мои заметки (сырые, как записывала на лету):
«Были: Сергей (CEO), Маша (HR), Антон (CTO), я. 
Обсуждали переезд в новый офис. Сергей: переезжаем 1 июня,
бюджет 2 млн, Антон отвечает за IT-инфраструктуру в новом офисе,
нужно до 15 мая закупить мебель. Маша: надо предупредить 
сотрудников минимум за месяц, спросить кто хочет помочь.
Решили: объявить 1 мая, Маша пишет рассылку, Антон делает план 
переезда серверов к 20 мая. Я — координирую мебель. 
Следующая встреча 5 мая.»

Формат протокола:
1. Дата, время, участники
2. Повестка (1 строка)
3. Обсуждение (ключевые тезисы, не стенограмма)
4. Решения (нумерованный список)
5. Задачи (таблица: задача | ответственный | срок)
6. Следующая встреча

Тон: деловой, лаконичный. Длина: 1 страница максимум.
Дата совещания: 16.04.2026, 14:00-14:45.
```

### Эталон 2: Объявление для сотрудников

```
Напиши объявление для корпоративного чата о переходе на новую 
систему бронирования переговорных.

Контекст:
- Компания: 60 человек, 3 переговорных
- Раньше: бронировали в Google Calendar, но постоянные конфликты
  (двойные бронирования, забывали отменить)
- Теперь: внедрили систему Meetly (экран у каждой переговорной)
- Старт: с понедельника 21 апреля
- Инструкция: в корпоративной wiki, ссылка wiki.company.ru/meetly
- Ключевое изменение: бронирование ТОЛЬКО через Meetly, 
  Google Calendar больше не работает для переговорных
- Если не забронировал — переговорная свободна для любого
- Максимум бронирования: 2 часа, продлить можно если никто не записан следом

Тон: дружелюбный, не формальный. Коротко (100-150 слов).
Добавить: 1 эмодзи-акцент + кого спросить при вопросах (меня, Ирина).
```

### Эталон 3: Проект регламента

```
Ты — опытный офис-менеджер. Составь регламент использования 
корпоративных автомобилей для компании.

Контекст:
- Компания: дистрибьютор оборудования, 50 сотрудников
- Автопарк: 3 автомобиля (2 легковых, 1 фургон)
- Водителей нет — сотрудники ездят сами
- Проблемы сейчас: никто не знает, свободна ли машина; возвращают
  с пустым баком; не фиксируют пробег; царапины появляются неизвестно когда

Регламент должен содержать:
1. Кто имеет право использовать автомобили (категории сотрудников)
2. Порядок бронирования (через кого, за какой срок)
3. Правила использования (заправка, чистота, пробег)
4. Что делать при ДТП / повреждении
5. Ответственность (штрафы, возмещение)
6. Приложение: форма заявки на использование

Тон: чёткий, но не угрожающий. 2 страницы максимум.
```

## 4. ТИПИЧНЫЕ ОШИБКИ

| # | Ошибка | Пример | Что делать |
|---|--------|--------|-----------|
| 1 | Формальный язык | «Доводим до вашего сведения» в чат | Указать тон: «как в рабочем чате, не как в приказе» |
| 2 | Нет контекста компании | Generic протокол без имён | Передать заметки + имена + должности |
| 3 | Протокол-стенограмма | Записал ВСЁ слово в слово | Указать: «ключевые решения и задачи, не стенограмма» |
| 4 | ИИ-фактология | ИИ дописал «согласно Трудовому кодексу ст. 99» | Не ссылаться на нормативы без проверки |
| 5 | Без формата | «Сделай объявление» | Указать: канал, длину, тон, ЦА, CTA |
| 6 | Персональные данные | Загрузил паспортные данные сотрудников | Корпоративный API или обезличивание |

## 6. ПРОФИЛЬНЫЕ КРИТЕРИИ КАЧЕСТВА РЕЗУЛЬТАТА

### Для задачи «Протокол»:
- Структура полная (участники, решения, задачи): 0-3
- Задачи конкретны (ответственный + срок): 0-3
- Лаконичность (не стенограмма): 0-2
- Тон деловой, но читаемый: 0-2
- **Максимум: 10**

### Для задачи «Объявление / рассылка»:
- Тон соответствует каналу: 0-2
- Ключевая информация на виду (что, когда, что делать): 0-3
- Длина адекватная: 0-2
- Есть CTA / контакт для вопросов: 0-1
- Не формальный «канцелярский»: 0-2
- **Максимум: 10**

### Для задачи «Регламент»:
- Структура полная: 0-2
- Практичность (можно применить завтра): 0-3
- Конкретные правила, не абстракции: 0-3
- Есть приложения (формы, чек-листы): 0-2
- **Максимум: 10**
