# ПУЛ ЗАДАНИЙ: МЕНЕДЖЕР ПО ПРОДАЖАМ (B2B)

> **Профиль:** sales_b2b
> **Всего заданий:** 7 (2 базовых + 3 средних + 2 продвинутых)
> **Лимит времени Sandbox:** 45 минут
> **Ротация:** система выдаёт 1 задание случайно из пула соответствующего уровня

---

## УРОВЕНЬ: БАЗОВЫЙ

> Для сотрудников с Этап 1 < 75% или промптинг < 60%.
> Одна задача, один промпт, минимальный контекст.

### ЗАДАНИЕ B-1: Follow-up письмо после звонка

**Контекст:**
Вы — менеджер по продажам в компании, которая продаёт облачную телефонию для бизнеса. Вчера вы провели телефонный разговор с потенциальным клиентом.

**Вводные данные (прикреплены к заданию):**
```
Клиент: Марина Сергеевна, руководитель отдела продаж, ООО «Мебель-Опт»
Компания: оптовая продажа мебели, 30 сотрудников, Нижний Новгород
Разговор: 20 мин. Обсудили проблемы — менеджеры пропускают звонки,
нет записи разговоров, CRM не интегрирована с телефонией.
Марина заинтересована, но хочет сравнить с текущим провайдером.
Попросила прислать: тарифы, кейс из мебельной отрасли, срок подключения.
Следующий шаг: она обсудит с директором и вернётся через неделю.
```

**Задание:** С помощью ИИ напишите follow-up письмо Марине Сергеевне. Письмо должно закрепить договорённости, ответить на её запросы и мотивировать к следующему шагу.

**Формат результата:** Готовое письмо (тема + тело), 150-250 слов.

**Бенчмарк ручной работы:** 25 минут

**Что оценивает Judge:**
- Промптинг: дал ли сотрудник контекст из вводных данных в промпт?
- Результат: есть ли привязка к разговору, конкретика, следующий шаг?
- Итерации: доработал ли или отправил первый вариант?

---

### ЗАДАНИЕ B-2: Ответы на типовые возражения

**Контекст:**
Вы — менеджер по продажам в компании, которая продаёт сервис электронного документооборота (ЭДО) для малого бизнеса.

**Вводные данные:**
```
Продукт: облачный сервис ЭДО, подписка 2 900 ₽/мес за 10 пользователей
Целевая аудитория: малый бизнес, 10-50 сотрудников
Конкуренты: Диадок, 1С-ЭДО, СБИС
Преимущества: проще в настройке, бесплатная интеграция с 1С, 
техподдержка 24/7, бесплатный пробный период 14 дней
```

**Задание:** С помощью ИИ подготовьте ответы на 3 возражения, которые вы чаще всего слышите от клиентов:
1. «У нас уже есть Диадок, зачем переходить?»
2. «2 900 в месяц — дорого для нас»
3. «Мы пока не готовы, может позже»

**Формат результата:** Для каждого возражения: ответ менеджера (2-3 предложения) + вопрос, переводящий разговор дальше.

**Бенчмарк ручной работы:** 30 минут

---

## УРОВЕНЬ: СРЕДНИЙ

> Для сотрудников с Этап 1 ≥ 75% и промптинг ≥ 60%.
> Задача требует 2-3 итерации, развёрнутый бриф.

### ЗАДАНИЕ M-1: Коммерческое предложение из ТЗ

**Контекст:**
Вы — менеджер по продажам в IT-компании. Клиент прислал запрос на внедрение CRM-системы. Вам нужно подготовить коммерческое предложение.

**Вводные данные (бриф от клиента — прикреплён):**
```
Компания: ООО «ПромТех», оптовая торговля стройматериалами
Город: Екатеринбург
Сотрудников: 80 (из них отдел продаж — 12 менеджеров)
Текущая ситуация: менеджеры ведут клиентов в Excel-таблицах.
Нет единой базы, заявки теряются, руководитель не видит воронку.
Запрос: внедрить CRM, интегрировать с 1С Бухгалтерия,
настроить автоматические отчёты по воронке продаж.
Бюджет: до 500 000 ₽ (внедрение + первый год лицензий)
Срок: хотят запуститься за 2 месяца
Принимает решение: коммерческий директор + генеральный
Конкуренты: получили ещё 2 предложения от других интеграторов
```

**Задание:** С помощью ИИ составьте коммерческое предложение, которое можно отправить клиенту. Включите: понимание задачи, решение, этапы, сроки, стоимость, гарантии, следующий шаг.

**Формат результата:** Готовый документ КП, 2-3 страницы.

**Бенчмарк ручной работы:** 90 минут

**Что оценивает Judge дополнительно:**
- Использовал ли сотрудник ВЕСЬ контекст из брифа (боль, бюджет, конкуренты)?
- Есть ли персонализация под отрасль (стройматериалы)?
- Адекватна ли разбивка стоимости?
- Было ли минимум 2 итерации (доработки промпта)?

### ЗАДАНИЕ M-2: Подготовка к встрече с клиентом

**Контекст:**
Завтра у вас первая встреча с потенциальным клиентом. Вам нужно подготовиться: изучить клиента и составить план встречи.

**Вводные данные:**
```
Клиент: ООО «ФудПроф», производство и доставка готовой еды
для офисов, B2B-модель. Москва, 120 сотрудников.
Встреча с: Алексей Новиков, директор по развитию.
Источник лида: оставил заявку на сайте, написал «Интересует 
автоматизация приёма заказов и маршрутизации курьеров».
Ваш продукт: платформа автоматизации логистики и заказов.
Цена: от 50 000 до 200 000 ₽/мес в зависимости от объёма.
Информация из открытых источников: у клиента есть сайт 
foodprof.ru, instagram, отзывы на Яндексе (4.2 из 5).
```

**Задание:** С помощью ИИ подготовьте:
1. Краткий профиль клиента (что за компания, боли, потребности)
2. 5-7 вопросов для первой встречи (discovery questions)
3. 3 гипотезы: какие задачи клиента ваш продукт может решить
4. Возможные возражения и как на них ответить

**Формат результата:** Структурированный документ (1-2 страницы).

**Бенчмарк ручной работы:** 60 минут

### ЗАДАНИЕ M-3: Реанимация «уснувшей» сделки

**Контекст:**
У вас в CRM есть сделка, которая «зависла» 3 недели назад. Клиент перестал отвечать после получения вашего КП.

**Вводные данные:**
```
Клиент: Дмитрий Волков, собственник, ООО «АвтоПарк-Сервис»
(сеть автосервисов, 5 точек, 45 сотрудников, Самара)
Продукт: CRM для автосервисов
История: 
- 4 недели назад: входящая заявка с сайта
- 3.5 недели назад: квалификационный звонок (30 мин), 
  Дмитрий заинтересован, просил КП
- 3 недели назад: отправлено КП на 180 000 ₽/год
- Дмитрий открыл письмо (по данным трекинга), но не ответил
- 2 недели назад: вы позвонили, не дозвонились
- 1 неделя назад: написали в WhatsApp «Дмитрий, добрый день!
  Удалось ли посмотреть наше предложение?» — прочитано, нет ответа
```

**Задание:** С помощью ИИ подготовьте стратегию реанимации этой сделки:
1. Анализ: почему клиент мог «замолчать» (3-5 гипотез)
2. Цепочка из 3 касаний (email + WhatsApp + звонок) с текстами
3. «Последнее» письмо-разрыв, если не ответит

Каждое сообщение должно быть готово к отправке.

**Формат результата:** Документ со стратегией и готовыми текстами.

**Бенчмарк ручной работы:** 45 минут

## УРОВЕНЬ: ПРОДВИНУТЫЙ

> Для сотрудников с Этап 1 ≥ 85% и промптинг ≥ 75%.
> Сложный бриф, требуется пайплайн из нескольких шагов.

### ЗАДАНИЕ A-1: Полный цикл — от ресёрча до КП

**Контекст:**
Вы получили лид с конференции. Визитка + короткая записка от коллеги. Нужно за один сеанс пройти весь цикл: ресёрч → подготовка → КП → сопроводительное письмо.

**Вводные данные:**
```
Визитка:
Елена Павлова
Директор по маркетингу
ООО «ТурбоТревел» — онлайн-агрегатор туров
Москва | elena@turbotravel.ru

Записка коллеги:
«Встретил на конференции. Компания растёт, 200+ сотрудников.
Жалуется, что отдел продаж (20 чел) работает вразнобой — 
у каждого свой Excel, общей воронки нет. Потенциал большой.
Елена сказала — пришлите что-нибудь, посмотрю.»

Ваш продукт: платформа CRM + аналитики для travel-индустрии.
Цена: от 300 000 до 800 000 ₽/год в зависимости от пакета.
```

**Задание:** С помощью ИИ выполните полный цикл подготовки:
1. Составьте профиль компании и лица, принимающего решения (на основе имеющихся данных)
2. Сформулируйте 3 ключевые гипотезы — какие боли вы можете решить
3. Подготовьте персонализированное КП (2 страницы)
4. Напишите сопроводительное email-письмо к КП

**Важно:** Покажите весь процесс — как вы используете ИИ на каждом шаге. Промпты должны быть связаны между собой (результат предыдущего = вход для следующего).

**Формат результата:** 4 документа (профиль + гипотезы + КП + письмо) или один сводный.

**Бенчмарк ручной работы:** 120 минут

**Что оценивает Judge дополнительно:**
- Есть ли пайплайн (каждый шаг использует результат предыдущего)?
- Промпты ссылаются друг на друга или каждый с нуля?
- Финальный результат — цельный и консистентный?

### ЗАДАНИЕ A-2: Стратегия работы с портфелем клиентов

**Контекст:**
Руководитель попросил вас подготовить аналитику по вашему портфелю клиентов и предложить стратегию на квартал.

**Вводные данные:**
```
Ваш портфель: 25 активных клиентов

Сегмент A (5 клиентов): Крупные, LTV > 1 млн/год.
  Средний чек 120 000 ₽/мес. Допродажи возможны.
  Один клиент намекает на уход к конкуренту.

Сегмент B (12 клиентов): Средние, LTV 300-800 тыс/год.
  Стабильные, но не растут. Потенциал для upsell.

Сегмент C (8 клиентов): Мелкие, LTV < 200 тыс/год.
  Много сервисных запросов, маржа низкая.
  2 клиента задерживают оплату.

Ваш план на квартал: 3 500 000 ₽ (новые + текущие).
Текущий прогноз: 2 800 000 ₽ (дефицит 700 000 ₽).
```

**Задание:** С помощью ИИ подготовьте:
1. Анализ портфеля: риски и возможности по каждому сегменту
2. Стратегию закрытия дефицита 700 000 ₽ (конкретные действия)
3. План по удержанию клиента из сегмента A (который может уйти)
4. Рекомендацию по сегменту C (что делать с убыточными клиентами)
5. Недельный план действий на ближайший месяц

**Формат результата:** Аналитическая записка для руководителя (2-3 страницы).

**Бенчмарк ручной работы:** 90 минут
