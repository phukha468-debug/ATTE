## УРОВЕНЬ: БАЗОВЫЙ

### ЗАДАНИЕ B-1: Ответ на жалобу о задержке доставки

**Контекст:** Вы — специалист поддержки интернет-магазина.

**Вводные данные:**
```
Клиент: Марина Петрова
Канал: email
Тема: «Где мой заказ?!»
Сообщение клиента: «Сделала заказ 5 дней назад (№58291), обещали доставить 
за 2 дня. Сегодня должна была забрать, звонила курьеру — он не в курсе! 
Это безобразие! Верните деньги!»

Данные по заказу:
- Сумма: 8 450 ₽ (набор кастрюль и сковородок)
- Статус: передан курьерской службе 3 дня назад
- Фактическая причина задержки: ошибка сортировки на складе курьерки
- Текущий статус: доставка завтра до 14:00 (подтверждено курьеркой)
- Полномочия менеджера: промокод 10% на следующий заказ (до 1000 ₽)
```

**Задание:** С помощью ИИ напишите ответ клиентке. Извинение + честная причина + новый срок + компенсация.

**Формат:** Email-ответ 120-180 слов. **Бенчмарк:** 20 мин.

### ЗАДАНИЕ B-2: Ответ на технический вопрос

**Контекст:** Вы — специалист поддержки облачного сервиса хранения файлов.

**Вводные данные:**
```
Клиент: Дмитрий, малый бизнес-владелец
Канал: чат на сайте
Сообщение: «Подскажите, как загрузить папку с файлами целиком? У меня 
150 документов, по одному загружать долго. Не нашёл в интерфейсе.»

Факты о продукте:
- Загрузка папок поддерживается только в десктоп-приложении (не в веб)
- В веб-версии можно выделить множество файлов (Ctrl+A в окне выбора)
- Десктоп-приложение доступно для Windows и Mac, бесплатно
- Ссылка на скачивание: app.cloudfile.ru/download
```

**Задание:** С помощью ИИ напишите ответ. Предложите два решения (десктоп-приложение + мультивыбор в вебе), укажите плюсы каждого.

**Формат:** Ответ в чате 100-150 слов. **Бенчмарк:** 15 мин.

## УРОВЕНЬ: СРЕДНИЙ

### ЗАДАНИЕ M-1: Создание раздела FAQ

**Контекст:** Компания запускает новый раздел в базе знаний. Нужна FAQ по популярной теме.

**Вводные данные:**
```
Продукт: SaaS-сервис онлайн-записи для сферы услуг (салоны, клиники)
Тарифы: Базовый (1 500 ₽/мес), Pro (3 500 ₽/мес), Enterprise (индивидуально)

Тема FAQ: «Напоминания клиентам о записи»

Что вы знаете о функциональности:
- SMS-напоминания: доступны на всех тарифах
- Email-напоминания: доступны на всех тарифах
- WhatsApp-напоминания: только Pro и выше
- Настройка интервалов: за 24 ч / 2 ч / оба варианта
- Кастомизация текста: только Pro (в базовом — только шаблон)
- Статистика доставки: все тарифы
- Лимит SMS: 500/мес в Базовом, безлимит в Pro
```

**Задание:** С помощью ИИ создайте FAQ из 8 вопросов, покрывающих весь путь:
- Базовые: «Как включить напоминания?»
- Средние: «Могу ли я настроить свой текст напоминания?»
- Продвинутые: «Как настроить разные напоминания для разных услуг?»

Для каждого: формулировка вопроса (естественная), короткий ответ, развёрнутый ответ, связанные вопросы, ограничения тарифа (если есть).

**Формат:** Готовый раздел FAQ. **Бенчмарк:** 60 мин.

### ЗАДАНИЕ M-2: Разбор эмоциональной жалобы

**Контекст:** Поступила гневная жалоба от постоянного клиента. Случай непростой.

**Вводные данные:**
```
Клиент: Александр Сергеевич, предприниматель, клиент банка 6 лет, VIP-статус
Канал: email + копия директору

Сообщение клиента (сокращённо):
«Ваш банк — позор! Пользуюсь вами 6 лет, а сегодня ваш сотрудник 
в отделении ХАМИЛ мне в лицо, когда я попросил помочь с платежом 
за оборудование для бизнеса. Сказала: "я не обязана вам помогать 
с корпоративными переводами, идите в банк-клиент сами". Я потерял 
день, контрагент ждёт денег, сделка может сорваться! Немедленно 
требую официальных извинений и перевести меня в другой банк бесплатно 
с полной компенсацией комиссий за последние полгода!»

Факты:
- Жалоба на сотрудника подтвердилась (записи с видеокамер есть)
- Сотрудница была некорректна, но платёж действительно можно было 
  провести только через банк-клиент (система так настроена)
- Клиент действительно VIP: среднемесячный остаток 2+ млн
- Что мы можем: принести официальные извинения (управляющий отделения лично),
  выплатить компенсацию морального вреда 50 000 ₽, 
  выделить персонального менеджера, провести внутреннее разбирательство
- Что НЕ можем: перевести в другой банк (не наша компетенция), 
  вернуть все комиссии (только за месяц по политике)
```

**Задание:** С помощью ИИ подготовьте:
1. Ответ клиенту (первый шаг — снять напряжение, признать, назначить встречу)
2. Внутреннюю записку руководителю отделения (что произошло, что предложить)

**Формат:** Два документа. **Бенчмарк:** 45 мин.

Технические проблемы (38 обращений):

- «Не могу войти в личный кабинет» — 12

Качество обучения (31):
- «Ребёнку не нравится преподаватель» — 14
- «Слишком сложно/скучно» — 11
- «Хочу сменить расписание» — 6

Оплата (22):
- «Списали деньги после отмены подписки» — 10

3. Конкретные рекомендации для каждого приоритета (что сделать команде продукта / техподдержке / преподавателям)

**Формат:** Отчёт для руководителя (2 страницы). **Бенчмарк:** 45 мин.

## УРОВЕНЬ: ПРОДВИНУТЫЙ

### ЗАДАНИЕ A-1: Построение базы знаний для продукта

**Контекст:** Компания запускает новый продукт. Нужна полная база знаний с нуля — для клиентов и внутренней поддержки.

**Вводные данные:**
```
Продукт: платформа для автоматизации рассылок SMS/WhatsApp/Email для МСБ
Тарифы: Старт (990 ₽/мес), Рост (2 900 ₽/мес), Бизнес (9 900 ₽/мес)

Ключевые функции:
- Настройка кампаний (одноразовых и триггерных)
- Сегментация базы (теги, поведенческие критерии)
- A/B-тестирование (только Рост+)
- Интеграции с CRM (amoCRM, Bitrix24 — все тарифы; МойСклад — Рост+)
- Аналитика (базовая во всех, продвинутая в Бизнес)
- Белые списки / чёрные списки
- Лимиты: Старт — 1000 контактов, 5000 сообщений/мес; Рост — 10000/50000;
  Бизнес — без лимитов + выделенная линия

Типичные пользователи:
- Собственники МСБ без маркетологов (70%)
- Маркетологи агентств (20%)
- Крупные корпоративные пользователи (10%)

У вас есть: техническая документация, но нет пользовательской базы знаний.
```

**Задание:** С помощью ИИ создайте структуру и наполнение базы знаний:
1. Верхнеуровневая структура (6-10 разделов)
2. Для каждого раздела — список статей (3-8 статей) с краткими описаниями
3. 3 детально написанные статьи (одна для новичка, одна средняя, одна продвинутая)
4. Глоссарий терминов (10-15 ключевых понятий с определениями)

**Формат:** Полный документ базы знаний. **Бенчмарк:** 150 мин.

### ЗАДАНИЕ A-2: Стратегия мультиканальной поддержки

**Контекст:** Руководитель попросил подготовить стратегию развития поддержки — растём в 3 раза за год, нужно масштабировать.

**Вводные данные:**
```
Компания: сервис облачной бухгалтерии для МСБ

Текущее состояние:
- Клиентов: 4 500 компаний
- Специалистов поддержки: 5 человек
- Каналы: email (60% обращений), чат на сайте (30%), телефон (10%)
- SLA: email 24ч, чат 10 мин, телефон ответ на 3-й гудок
- CSAT: 78% (цель 85%+)
- Типичные темы: регистрация (20%), настройка интеграций (25%), 
  вопросы по отчётности (30%), технические сбои (15%), прочее (10%)

Планы на год:
- Рост до 13 500 клиентов (×3)
- Бюджет на расширение поддержки: +8 млн ₽/год
- Приоритеты руководителя: сохранить CSAT, снизить нагрузку на команду,
  не раздувать штат пропорционально росту клиентов
```

**Задание:** С помощью ИИ подготовьте стратегию:
1. Анализ текущего состояния (узкие места, риски при росте)
2. Рекомендации по каналам: что оставить, что добавить, что убрать
3. План автоматизации (с помощью ИИ + базы знаний): какие % вопросов можно закрыть без человека
4. Оптимальный состав команды через год (роли, количество, распределение)
5. Метрики для отслеживания + целевые значения
6. Поэтапный план внедрения (0-3 / 3-6 / 6-12 месяцев)

**Формат:** Стратегический документ (3-4 страницы). **Бенчмарк:** 120 мин.
