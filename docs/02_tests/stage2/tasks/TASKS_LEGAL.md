# ПУЛ ЗАДАНИЙ: ЮРИСТ / ЮРИСКОНСУЛЬТ

> **Профиль:** legal | **Заданий:** 7 (2+3+2) | **Лимит:** 45 мин

---

## УРОВЕНЬ: БАЗОВЫЙ

### ЗАДАНИЕ B-1: Разъяснение для сотрудника

**Вводные данные:**
```
Сотрудник (менеджер по продажам) написал вам:
«Клиент просит подписать NDA перед тем, как мы покажем ему демо 
нашей платформы. Он прислал свой шаблон. Я могу подписать или 
нужно через вас? И вообще, что такое NDA?»

Контекст:
- Компания: SaaS-платформа для управления складами
- Обычно: НДА подписывает директор или юрист по доверенности
- У менеджера нет права подписи
- NDA клиента — стандартный, вы его просмотрели, замечаний нет
```

**Задание:** С помощью ИИ подготовьте ответ: объясните что такое NDA, кто подписывает, что делать дальше. Понятным языком.

**Формат:** Сообщение в чат 150-200 слов. **Бенчмарк:** 15 мин.

---

### ЗАДАНИЕ B-2: Письмо контрагенту о расторжении договора

**Вводные данные:**
```
Компания: маркетинговое агентство
Ситуация: расторгаем договор с подрядчиком (фрилансер-дизайнер)
Причина: систематические срывы сроков (3 из 5 задач — с опозданием 5+ дней)
Договор: ГПХ, пункт 7.2 — «любая сторона может расторгнуть 
с уведомлением за 14 дней»
Оплата: всё оплачено за выполненные работы, долгов нет
Тон: корректный, без обвинений, сохранить нормальные отношения
```

**Задание:** С помощью ИИ напишите уведомление о расторжении.

**Формат:** Деловое письмо 100-150 слов. **Бенчмарк:** 20 мин.

## УРОВЕНЬ: СРЕДНИЙ

### ЗАДАНИЕ M-1: Проект договора оказания услуг

**Вводные данные:**
```
Исполнитель (мы): ООО «ДиджиталПро», digital-агентство, Москва
Заказчик: ООО «СтройГрупп», строительная компания
Предмет: ведение контекстной рекламы (Яндекс.Директ) на 3 месяца

Условия:
- Стоимость: 120 000 ₽/мес (ведение) + рекламный бюджет клиента отдельно
- Оплата: предоплата за месяц до 5-го числа
- KPI: мы гарантируем не менее 150 заявок/мес при бюджете от 200 000 ₽
- Отчётность: еженедельный отчёт по пятницам
- Расторжение: за 30 дней, с оплатой за отработанный период
- Наши риски: клиент может менять сайт/посадочные без согласования,
  что обрушит конверсию — нужен защитный пункт

Структура: предмет → обязанности → оплата → отчётность → 
KPI и ответственность → конфиденциальность → расторжение → споры

НЕ ссылаться на конкретные статьи ГК. Писать «в соответствии 
с действующим законодательством РФ», нормы добавлю сам.
```

**Задание:** С помощью ИИ подготовьте проект договора.

**Формат:** Договор 3-4 страницы. **Бенчмарк:** 60 мин.

### ЗАДАНИЕ M-2: Юридическая справка для руководителя

**Вводные данные:**
```
Вопрос директора: «Мы хотим перевести часть сотрудников на удалёнку 
на постоянной основе. Что нужно оформить юридически? Какие риски?»

Контекст компании:
- IT-компания, 45 сотрудников, все по ТК (трудовые договоры)
- 15 сотрудников хотят на полную удалёнку
- 10 — на гибрид (2 дня офис / 3 удалённо)
- Остальные 20 — в офисе
- Офис: аренда в БЦ, 250 м² (можно уменьшить до 150 м² и сэкономить)
- Техника: ноутбуки у всех корпоративные

Что нужно осветить:
1. Какие документы оформить (допсоглашения, приказы, положение)
2. Особенности охраны труда на удалёнке
3. Контроль рабочего времени — что допустимо, что нет
4. Налоговые последствия (если кто-то переедет в другой регион)
5. Риски и как их минимизировать

Написать для не-юриста (директор — технарь).
НЕ ссылаться на конкретные статьи ТК — я добавлю сам.
```

**Задание:** С помощью ИИ подготовьте справку (1.5-2 страницы).

**Формат:** Правовая справка. **Бенчмарк:** 45 мин.

### ЗАДАНИЕ M-3: Досудебная претензия

**Вводные данные:**
```
Ситуация:
- Наша компания: ООО «АвтоЛогистик» (транспортные услуги)
- Контрагент: ООО «ПромСнаб» (заказчик перевозки)
- Договор: №78 от 20.01.2026, перевозка оборудования
- Стоимость: 340 000 ₽
- Оплата: 100% по факту доставки, срок 5 рабочих дней после подписания акта
- Акт подписан: 15.02.2026
- Оплата получена: НЕТ (на 17.04.2026 — 43 рабочих дня просрочки)
- Напоминания: 2 письма (01.03, 25.03) + 3 звонка — обещают, не платят
- Неустойка по договору: 0.1% от суммы за каждый день просрочки
- Расчёт неустойки: 340 000 × 0.001 × 43 = 14 620 ₽

Требуем:
1. Оплатить основной долг 340 000 ₽ в течение 10 календарных дней
2. Уплатить неустойку 14 620 ₽ (на дату претензии)
3. В случае неисполнения — обращаемся в арбитражный суд

Тон: строгий, без эмоций. Юридически корректно, но без 
копирования конкретных статей ГК (я проверю и добавлю).
```

**Задание:** С помощью ИИ подготовьте досудебную претензию.

**Формат:** Претензия 1-1.5 страницы. **Бенчмарк:** 30 мин.

## УРОВЕНЬ: ПРОДВИНУТЫЙ

### ЗАДАНИЕ A-1: Правовой аудит типового договора

**Вводные данные:**
```
Клиент прислал нам свой шаблон договора на подписание. 
Нужно проанализировать его на риски для нашей стороны.

Ваша компания (исполнитель): IT-интегратор, внедряет CRM
Контрагент (заказчик): крупная строительная компания

Ключевые пункты их договора, вызывающие сомнения:

П. 3.2: «Исполнитель обязан обеспечить бесперебойную работу системы 
24/7/365 с уровнем доступности 99.99%»
П. 4.5: «Оплата производится в течение 90 рабочих дней после 
подписания акта»
П. 5.1: «Штраф за каждый день просрочки — 1% от стоимости договора»
П. 6.3: «Все исключительные права на результат переходят Заказчику 
с момента начала работ»
П. 7.1: «Заказчик вправе расторгнуть договор в любой момент 
без объяснения причин и компенсации»
П. 8.2: «Исполнитель несёт полную ответственность за убытки 
Заказчика, включая упущенную выгоду, без ограничений»
П. 9.1: «Споры рассматриваются в суде по месту нахождения Заказчика»

Стоимость договора: 3 200 000 ₽
Срок работ: 4 месяца
```

**Задание:** С помощью ИИ подготовьте:
1. Анализ рисков по каждому пункту (что плохо, почему)
2. Предложение по каждому пункту (наша редакция)
3. Протокол разногласий (таблица: пункт → их редакция → наша → обоснование)
4. Стратегия переговоров: от чего не отступать, где можно пойти на компромисс

**Формат:** Аналитический документ (3-4 страницы). **Бенчмарк:** 90 мин.

### ЗАДАНИЕ A-2: Политика обработки персональных данных

**Вводные данные:**
```
Компания: онлайн-платформа для бронирования спортивных площадок
Размер: 25 сотрудников, B2C-модель
Пользователей: 12 000 зарегистрированных

Какие ПД собираем:
- Клиенты: ФИО, телефон, email, дата рождения, фото (опционально),
  история бронирований, платёжные данные (обрабатывает Яндекс.Касса)
- Сотрудники: стандартный кадровый комплект
- Партнёры (площадки): ФИО контактных лиц, телефоны, email

Каналы сбора: сайт, мобильное приложение, call-центр
Хранение: серверы в РФ (Яндекс.Облако)
Передача третьим лицам: Яндекс.Касса (платежи), SMS-провайдер (уведомления)

Что нужно:
1. Политика обработки ПД (для сайта — должна быть публичной)
2. Согласие на обработку ПД (форма для пользователя)
3. Чек-лист для компании: что ещё нужно сделать по 152-ФЗ
```

**Задание:** С помощью ИИ подготовьте все 3 документа.

**Формат:** 3 документа (политика 3-4 страницы, согласие 1 страница, чек-лист). **Бенчмарк:** 120 мин.

**Важно:** документы — черновики для юридической проверки. НЕ ссылаться на конкретные пункты 152-ФЗ без проверки.
