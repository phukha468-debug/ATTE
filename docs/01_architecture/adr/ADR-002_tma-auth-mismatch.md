# ADR-002: TMA Auth Hydration Mismatch
**Дата:** 2026-04-17
**Статус:** accepted

## Контекст
При переключении аккаунтов Telegram на одном мобильном устройстве сессия пользователя "залипает" на первом аккаунте. Проблема отсутствует в Web и Desktop версиях.

## Первопричина
Мобильный WebView (TMA) разделяет `localStorage` между разными аккаунтами внутри одного клиента Telegram. Supabase автоматически гидрирует сессию из `localStorage` до того, как приложение проверяет актуальный `initData` от Telegram SDK.

## Решение
На этапе загрузки (Boot) приложение обязано сравнивать `Telegram.WebApp.initDataUnsafe.user.id` с `userId` из текущей сессии Supabase. 
Если они не совпадают (mismatch) -> принудительный `supabase.auth.signOut()` -> очистка глобального стейта -> новая авторизация через актуальный `initData`.

## Антипаттерны
ЗАПРЕЩЕНО полагаться исключительно на наличие сессии Supabase без сверки с Telegram SDK.
