# АРХИТЕКТУРА ЭТАПА 2: СИМУЛЯТОР

> **Версия:** 1.0 | **Статус:** Согласованная концепция
> **Цель:** Проверить на практике, умеет ли сотрудник экономить время с помощью ИИ
> **Главный продукт:** Не баллы, а карта возможностей + конкретные рекомендации

## 1. МЕСТО ЭТАПА 2 В ВОРОНКЕ

```
ЭТАП 1                    ЭТАП 2                    ЭТАП 3
Карта рутины              Навыки в деле             Микро-проект
─────────────────────────────────────────────────────────────────

Выявляем:                 Проверяем:                Внедряем:
• Какие задачи рутинные   • Умеет ли использовать   • Реальная автоматизация
• Где видит потенциал       ИИ на практике          • Измеримый результат
• Базовое понимание       • Какое ускорение даёт    • ROI для руководителя
  промптинга              • Где зоны роста

     ──────────────>           ──────────────>
     Передаёт:                 Передаёт:
     • Карту рутины            • Баллы по 5 измерениям
     • Баллы по доменам        • Зоны роста
     • Уровень сложности       • Валидированная экономия
       для задания             • Рекомендации для
                                 микро-проекта
```

### Принцип накопления данных:

Каждый этап не просто оценивает — он **уточняет** картину.

**Этап 1 говорит:** «Сотрудник считает, что может сэкономить 6 ч/нед на КП.»
**Этап 2 проверяет:** «На практике его ускорение 3x. Реальная экономия — 4 ч/нед.»
**Этап 3 внедряет:** «Создан шаблон КП. Фактическая экономия за 2 недели — 3.5 ч/нед. Подтверждено.»

Итоговый отчёт содержит ВСЕ три слоя: заявленное → проверенное → внедрённое.

## 3. ДВОЙНАЯ ИИ-АРХИТЕКТУРА

```
┌──────────────────────┐    ┌──────────────────────┐
│   ИИ-АССИСТЕНТ       │    │   ИИ-СУДЬЯ (Judge)   │
│   (Sandbox)          │    │                      │
├──────────────────────┤    ├──────────────────────┤
│ Роль: нейтральный    │    │ Роль: оценщик        │
│ помощник              │    │                      │
│                      │    │ Получает: весь лог    │
│ Контакт: прямой      │    │ переписки + задание   │
│ с сотрудником        │    │ + данные Этапа 1      │
│                      │    │                      │
│ Системный промпт:    │    │ Системный промпт:     │
│ «Ты — ИИ-ассистент.  │    │ Core Skill +          │
│  Выполняй запросы.   │    │ профильный модуль     │
│  Не подсказывай,     │    │                      │
│  не оценивай,        │    │ Контакт: НЕТ.         │
│  не направляй.»      │    │ Только анализ.        │
└──────────────────────┘    └──────────────────────┘
         ↓                            ↓
    Генерирует                  Генерирует
    ответы по запросу           оценку + обратную связь
    сотрудника                  + данные для отчёта
```

### Почему два ИИ, а не один:

**Чистота оценки.** Если ИИ одновременно помогает и оценивает — он невольно направляет сотрудника к лучшим ответам. Мы тестируем реальный навык, а не способность следовать подсказкам.

**Разные системные промпты.** Ассистент должен быть «тупым» — как обычный ChatGPT без инструкций. Judge должен быть «умным» — с рубриками, эталонами, критериями.

**Независимость.** Judge анализирует постфактум, видя полную картину. Он может оценить паттерн (первый промпт слабый, но сотрудник доработал — это хорошо), что невозможно в реальном времени.

## 4. МОДУЛЬНАЯ СИСТЕМА СКИЛЛОВ

```
skills/
├── SKILL_JUDGE_CORE.md          ← Ядро оценки (общее для всех)
│   • 5 измерений с рубриками
│   • Формула итогового балла
│   • Шаблон обратной связи
│   • Логика интеграции с Этапом 1
│
profiles/
├── PROFILE_SALES_B2B.md         ← Профильный модуль: Продажи B2B
├── PROFILE_MARKETING.md         ← Профильный модуль: Маркетинг
├── PROFILE_HR.md                ← Профильный модуль: HR
├── PROFILE_FINANCE.md           ← ... и так далее (20 профилей)
│   Каждый содержит:
│   • Типичная рутина должности (экспертная карта)
│   • ТОП-3 задачи для автоматизации
│   • Эталонные промпты-образцы
│   • Типичные ошибки
│   • Профильные критерии качества результата
│   • Метаданные
│
tasks/
├── TASKS_SALES_B2B.md           ← Пул заданий: Продажи B2B
├── TASKS_MARKETING.md           ← Пул заданий: Маркетинг
├── TASKS_HR.md                  ← ... и так далее
│   Каждый содержит:
│   • 5-7 заданий трёх уровней сложности
│   • Бриф/ТЗ для каждого задания
│   • Бенчмарки ручной работы
│   • Специфические критерии для Judge
```

### Почему модульная, а не монолитная:

1. **Обновление.** Поменял критерий оценки промптов → обновил 1 файл (Core), а не 20.
2. **Масштабирование.** Добавить новый профиль = написать 1 профиль + 1 пул заданий. Core не трогаем.
3. **Калибровка.** После стресс-тестов на 10 компаниях можно подкрутить профильные модули по отдельности — у бухгалтеров одни ошибки, у маркетологов другие.
4. **Прозрачность.** Каждый файл — самостоятельный документ, который можно показать клиенту или эксперту.

## 6. ЭКСПЕРТНЫЕ ПРОФИЛИ: ЧТО ОНИ ДАЮТ КЛИЕНТУ

Экспертный профиль — это не просто набор заданий. Это **экспертиза 66ai**, упакованная в продукт.

### Для сотрудника:
- Понимание, какие ЕГО задачи можно автоматизировать (не абстрактно, а конкретно)
- Примеры промптов для ЕГО работы
- Расчёт экономии в часах и рублях

### Для руководителя:
- Карта автоматизации отдела (какие задачи, сколько людей, сколько часов)
- Приоритизация: что автоматизировать в первую очередь
- Бенчмарки: сколько реально можно сэкономить

### Для 66ai:
- Актив, который улучшается с каждым стресс-тестом
- Конкурентное преимущество (у других тестирований нет отраслевой экспертизы)
- Основа для обучающих модулей

## 7. ПЛАН СТРЕСС-ТЕСТОВ (10 бесплатных компаний)

### Цель:
Калибровка профилей, рубрик Judge и заданий на реальных сотрудниках.

### Что получаем от каждой компании:

**До теста:**
- Состав отделов, роли, количество сотрудников
- Ожидания руководителя

**Во время теста:**
- Логи Sandbox (промпты + ответы)
- Время выполнения
- Результаты Judge

**После теста (интервью):**
- «Задание было реалистичным?» → калибровка заданий
- «Обратная связь была полезной?» → калибровка Judge
- «Что в вашей работе мы не покрыли?» → дополнение профиля
- «Какие задачи вы бы хотели автоматизировать, но не знали как?» → расширение ТОП-3

### Итерация:
```
Компания 1 → интервью → правки профилей и Judge →
Компания 2 → интервью → правки →
...
Компания 10 → финальная версия профилей
```

### Какие профили тестируем первыми:
Продажи B2B, Маркетолог, HR — на каждой компании хотя бы один из этих отделов.

## 8. ДАННЫЕ ДЛЯ ОТЧЁТОВ

### В отчёт сотрудника (из Этапа 2):
- Баллы по 5 измерениям
- Ускорение (Xx)
- Валидированная экономия (ч/нед, ₽/мес)
- Зоны роста + рекомендации
- Грейд (предварительный, финальный — после Этапа 3)

### В отчёт руководителя (агрегат по отделу):
- Средние баллы отдела по измерениям
- ТОП задач для автоматизации (из профилей + результаты сотрудников)
- Общий потенциал экономии отдела
- Кто может стать ИИ-чемпионом (грейд 4-5)
- Кому нужно обучение (грейд 1-2)

### В сводный отчёт компании (агрегат всех отделов):
- Карта автоматизации компании
- Рейтинг отделов
- Общий потенциал (часы/мес, ₽/мес, ₽/год)
- Приоритезированный план внедрения
