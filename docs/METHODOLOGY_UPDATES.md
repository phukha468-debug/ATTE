## SNAKE_EYES PROTOCOL

**Дата:** 14.04.2026
**Причина:** Ошибки 011/012 (`telegram_id` vs `tg_id`) выявили отсутствие защиты от рассинхрона схем.

### Правило

Любое обращение к БД требует предварительной сверки с физической схемой. Использование `camelCase` в именах колонок БД запрещено. Перед началом задач на DB-логику Исполнитель обязан проверить соответствие BRIEF и реальной таблицы.

### Источники правды

| Источник | Что содержит |
|----------|-------------|
| `docs/BRIEF.md` → раздел "Структура БД" | Единственный источник истины для схемы |
| `api/auth/telegram.ts` → `DB_SCHEMA` | Программная копия схемы, используется во всех запросах |
| `src/types/index.ts` | TypeScript-интерфейсы, должны совпадать с BRIEF |

### Запрещено

```
❌ .eq('telegram_id', ...)     → правильно: .eq('tg_id', ...)
❌ .insert({ fullName: ... })  → правильно: .insert({ full_name: ... })
❌ .eq('companyId', ...)       → правильно: .eq('company_id', ...)
❌ camelCase в именах колонок БД
❌ `any` в интерфейсах пользователя
```

### Разрешено

```
✅ Только snake_case для всех полей БД
✅ DB_SCHEMA константа для программных запросов
✅ TypeScript интерфейсы с snake_case полями (tg_id, full_name)
✅ Приоритет у BRIEF.md если код расходится с документацией
```

### Checklist перед DB-задачей

- [ ] Сверил имена колонок с `docs/BRIEF.md` → раздел "Структура БД"
- [ ] Проверил, что `DB_SCHEMA` в `api/auth/telegram.ts` актуальна
- [ ] Проверил, что TypeScript-интерфейсы совпадают с BRIEF
- [ ] Нет `camelCase` в `.eq()`, `.insert()`, `.select()` вызовах
