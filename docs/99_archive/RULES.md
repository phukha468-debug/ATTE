# ATTE — Правила разработки (RULES)

Свод правил для поддержания чистоты кода и стабильности системы.

---

## 🏗 Архитектура и Код
1. **Stability First:** Сначала логика и база данных, затем визуальное оформление (UI).
2. **Context Efficiency:** При внесении изменений сначала изучите существующие паттерны в `src/lib` и `src/components`.
3. **Types:** Все данные должны быть типизированы в `src/types` или локальных интерфейсах. Избегайте `any`.
4. **Mobile First:** Всегда проверяйте верстку на мобильных экранах (используйте `dvh` и `pb-safe`).

---

## 🛠 Технологии и Стек
- **Styles:** Предпочитайте **Vanilla CSS** и Tailwind v4. Избегайте избыточных библиотек.
- **State:** Глобальный стор — Zustand. Локальный стейт — React `useState`.
- **API:** Любая запись в БД через API должна сопровождаться блоком `try/catch` и обработкой ошибок.

---

## 🔒 Безопасность
- **Keys:** `VITE_SUPABASE_ANON_KEY` только для фронтенда. `SUPABASE_SERVICE_ROLE_KEY` — только для серверных функций (API).
- **Validation:** Входные данные от пользователя в API должны валидироваться.
- **Secrets:** Никогда не выводите секретные ключи в логи или на фронтенд.

---

## 📂 Структура файлов
- `src/components`: UI компоненты (shadcn — в `/ui`).
- `src/lib`: Логика, API-клиенты, утилиты.
- `src/pages`: Компоненты страниц.
- `api/`: Серверные функции Vercel.

---
*Соблюдение этих правил обязательно для всех участников разработки.*
