Пользователь выбрал профессию и прочитал бриф. Теперь он переходит в чат, где должен решить задачу с помощью ИИ. 
На этом шаге мы создаем UI чата и подключаем потоковый или обычный ответ от OpenRouter через новую Vercel Edge функцию. (Логика оценки "Судьей" будет реализована в следующем таске).

ЗАТРАГИВАЕМЫЕ ФАЙЛЫ: src/pages/SandboxPage.tsx (создать), src/App.tsx, api/ai/simulate.ts (создать)
ТИП: feat, ui, backend

<task>
1. **Routing & Navigation:**
   - Создать роут `/sandbox/:taskId` и страницу `SandboxPage.tsx`.
   - Оживить кнопку "Перейти в Sandbox" на экране брифинга (переход с передачей ID задачи).

2. **Chat UI (`SandboxPage.tsx`):**
   - Создать интерфейс чата в стиле Telegram (компактный, на всю высоту экрана, без горизонтального скролла).
   - Элементы: 
     - Хедер с названием задачи и таймером (отсчет времени с 00:00).
     - Область сообщений (бабблы "Вы" и "ИИ-Ассистент").
     - Поле ввода (textarea с авторесайзом) и кнопка отправки.
     - Индикатор загрузки "ИИ печатает...".
     - Большая кнопка "Завершить задачу" (пока просто выводит `alert('Здесь будет вызов Судьи')`).

3. **Backend API (`api/ai/simulate.ts`):**
   - Создать Edge-функцию для обработки диалога (`export const config = { runtime: 'edge' };`).
   - Функция принимает историю сообщений (`messages`) и возвращает ответ от OpenRouter (модель `openai/gpt-3.5-turbo` или `google/gemini-flash-1.5` для скорости).
   - Системный промпт ИИ: "Ты — полезный ИИ-ассистент корпоративной платформы. Твоя цель — четко и качественно выполнять рабочие поручения пользователя. Отвечай кратко и по делу."

4. **State Management:**
   - Хранить историю сообщений текущей сессии в локальном стейте React (`useState`).
   - При входе в чат автоматически добавлять первое системное сообщение от ИИ: "Привет! Я готов помочь с задачей. Что нужно сделать?"

5. **ВЕРИФИКАЦИЯ:**
   - Выбрать роль -> Открыть бриф -> Нажать "Перейти в Sandbox".
   - Написать сообщение в чат ("Привет, напиши письмо...") -> Получить адекватный ответ от ИИ.
   - Убедиться, что интерфейс чата не ломается при длинных текстах.
</task>
